# 08 Documentation Standards

Authority: `PROJECT_BIBLE.md`

## Documentation Rules

- Update documentation in the same phase that changes behavior, architecture, dependencies, or structure.
- Keep `PROJECT_BIBLE.md` as the authoritative source.
- Keep supporting docs focused and avoid repeating full standards.
- Add decisions to `DECISION_LOG.md`.
- Keep README practical for setup and navigation.

## Required Phase Handoff

Every phase handoff must state:

- Files created.
- Files modified.
- Dependencies installed or changed.
- Commands run.
- Quality gate result.
- Known warnings or issues.
- What is ready for the next phase.

## Assignment Docs

Assignment compliance must remain synchronized with `13_ASSIGNMENT_COMPLIANCE.md` and the Phase 1 traceability matrix.
