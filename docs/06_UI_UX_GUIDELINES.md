# 06 UI UX Guidelines

Authority: `PROJECT_BIBLE.md`

## Design Source

The final UI must follow the approved Figma design or an explicitly approved YUMMY design baseline. Until then, Phase 2 placeholder screens are not final design work.

## Customer UX

- Make ordering fast and obvious.
- Use food imagery where useful.
- Keep prices and actions visible.
- Provide clear cart totals.
- Show order status in plain language.

## Admin UX

- Prioritize scannable lists, status indicators, and quick actions.
- Avoid decorative layouts that slow repeated work.
- Use clear confirmation for destructive actions.

## Accessibility

Follow the accessibility standards in `PROJECT_BIBLE.md`. Any new screen must be checked for contrast, tap target size, labels, keyboard behavior, and status clarity.

## UI Change Control

When Figma becomes available, update this file only with project-specific decisions. Keep global rules in `PROJECT_BIBLE.md`.
